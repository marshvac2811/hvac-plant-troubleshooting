# HVAC Plant Troubleshooting Tool

A Python/Flask fault diagnosis reference and interactive rule engine for
HVAC plant equipment (chillers, AHUs, chilled water pumps, cooling
towers) — built directly from 26+ years of field HVAC/MEP troubleshooting
experience.

## Why this exists

Most troubleshooting knowledge in HVAC/MEP lives in an experienced
engineer's head — this project structures that knowledge into a
queryable, testable format: browse symptoms and causes for reference, or
enter actual readings into the interactive diagnostic engine and get a
live rule evaluation.

**The diagnostic content (symptoms, causes, required data, actions, and
rule logic) was authored by a domain expert with decades of field
experience — not generated by AI.** This tool only structures and
evaluates that expertise; it does not invent or supplement the
underlying diagnostic judgment.

## What it includes

### 1. Browse Symptoms
Reference lookup — select equipment (Chiller, AHU, Pump, Cooling Tower)
and a symptom to see possible causes, required sensor data, and
recommended actions.

### 2. Interactive Diagnosis
Seven explicit, quantifiable diagnostic rules with real AND-logic
conditions — enter actual readings (pressures, temperatures, yes/no
observations) and the tool evaluates whether the rule fires, e.g.:

> IF Discharge Pressure > HP Limit AND CW Entering Temp > 33°C
> THEN Cooling Tower Performance Poor

Covers: chiller high/low pressure trips, AHU dirty filter and water
leakage detection, and pump motor fault, bypass valve, and cavitation
diagnosis.

### 3. Fault Priority Matrix
Reference table classifying common faults by priority (Critical / High /
Medium / Low).

### 4. Sensor Validation Checker
Quick logical sanity checks for sensor readings (e.g., Return Air Temp
should always be greater than Supply Air Temp) — catches obviously wrong
sensor values before deeper diagnosis.

## Tech stack

- **Backend:** Python 3, Flask
- **Frontend:** Server-rendered Jinja2 templates (tabbed navigation), vanilla CSS
- **No database, no external services** — pure knowledge base + rule engine

## Getting started

```bash
git clone https://github.com/<your-username>/hvac-plant-troubleshooting.git
cd hvac-plant-troubleshooting
pip install -r requirements.txt
python app.py
```

Open **http://127.0.0.1:5008** in your browser.

## Project structure

```
hvac-plant-troubleshooting/
├── app.py                  # Flask routes for all 4 tools
├── knowledge_base.py          # Symptom/cause/action reference data
├── diagnostic_rules.py          # 7 interactive rule definitions + evaluation logic
├── requirements.txt
└── templates/
    ├── base.html                  # Shared layout + tab navigation
    ├── browse.html                 # Symptom/cause reference browser
    ├── diagnose.html                 # Interactive rule evaluator
    ├── priority.html                   # Fault priority matrix
    └── sensor_check.html                 # Sensor validation checker
```

## Roadmap / possible extensions

This project could grow toward a full Fault Detection and Diagnostics
(FDD) system: fault IDs, confidence scoring, severity ranking, ranked
probable causes, maintenance workflow integration, and historical
occurrence pattern tracking — suitable for implementation in a proper
rules engine or graph database for production BMS integration.
